Authorized Transaction

  • Description

    A Merchant-Initiated Merchant Payment is typically a received amount, where the Payer FSP is not disclosing any fees to the Payee FSP, in this test case has the focus to complete a successful transaction and approve the purchase.

  • Preconditions
    1. The Payer should have an account in chosen Mobile Money Operator 2.
    2. The Service Provider should have an account in chosen Mobile Money Operator 1.
  • Preconditions
    1. Configure your system to use the URL https://sandbox.mobilemoneyapi.io/v1.0/passthrough/mm/
    2. Configure your system to send a message to the TRANSACTION endpoint.
    3. Into the transaction body set the fields: - "contactPhone" to any value which ends on "1234", following the Mobile Money Specification - "amount" to the value "10.00", following the Mobile Money Specification
